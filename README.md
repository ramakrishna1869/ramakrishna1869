## What I do

I work at the seam between **building software and shipping it**. On one side I build full-stack
applications with the **MERN stack and Next.js** — server-rendered frontends, REST APIs, real-time
data, secure auth, and admin dashboards. On the other, I make that software **deployable and reliable**:
Jenkins and GitHub Actions pipelines, Docker images, Kubernetes deployments, and AWS infrastructure
(EC2, ECS, load balancers, IAM) with code quality and artifact gates wired in.

The part I care about is the gap most demos ignore — **how does this actually run in production?**
Consistent environments, high availability, secure access control, and pipelines that catch problems
before users do. That's where I spend my engineering effort.

## 🚀 What I build

- 🌐 **Full-Stack Web Apps** — End-to-end MERN & Next.js platforms: SSR frontends, Node/Express APIs, MongoDB data layers, secure login, and admin dashboards.
- 🔁 **CI/CD Pipelines** — Automated Jenkins & GitHub Actions pipelines that build, test, and deploy containerized services with quality gates and versioned artifacts.
- ☁️ **Cloud Deployments** — Production MERN/Next.js apps on AWS (EC2, ECS, Load Balancers, IAM) with Docker & Kubernetes for high availability and reliable scaling.
- 🧪 **Quality-Gated Delivery** — SonarQube for code quality and Nexus for artifact management, so every deploy is consistent, secure, and repeatable.
- 🧠 **Applied Machine Learning** — Classification models (Logistic Regression, Random Forest) with real feature engineering and proper evaluation (ROC, confusion matrices).
